## 🔍 三大根本原因

### 1. 数据域不匹配（60% 问题权重）

**OpenScore Lieder (5,238 图片) 的致命缺陷**：

| 问题类别 | 具体表现 | 影响 |
|---------|---------|------|
| **渲染风格差异** | Verovio 矢量图 vs 真实扫描图 | 模型学习两套冲突特征 |
| **类别极度不平衡** | 99.5% 标注集中在 4 类（fermata, barlines） | 其他 29 类"饥饿训练" |
| **标注不完整** | 缺少 stem, notehead, beam 等基础符号 | 负样本污染 |

**定量证据**：
- OpenScore 图像尺寸：1246x1749（较小）
- Phase 7 图像尺寸：未知（推测 >2000px）
- 风格差异导致 fermata 类别混淆，检测率下降

### 2. 标注质量问题（30% 问题权重）

**DeepScores Dynamics (700 图片) 的问题**：

| 问题 | 影响 |
|------|------|
| 尺寸过大（2107x2980） | Resize 到 640x640 损失 93.5% 像素信息 |
| 过度合成 | 缺少真实噪声，泛化能力差 |
| 类别单一 | 仅 dynamics，对整体 mAP50 贡献有限 |

**Phase 9 数据分布异质性**：
```
标注密度标准差: 208.1 (Phase 8: 12.4)
→ 16.8x 的异质性 = 梯度方向冲突 = 无法收敛
```

### 3. 训练配置不足（10% 问题权重）

| 配置 | Phase 8 | Phase 9 | 最优建议 |
|------|---------|---------|---------|
| epochs | 150 | 100 | 150-200 |
| lr0 | 0.001 | 0.001 (已修正) | ✅ 正确 |
| 训练策略 | 单阶段 | 单阶段 | ❌ 应分阶段 |

## 💡 关键数据科学洞察

### 洞察 1: 负迁移的量化证据

```python
# Phase 8: 健康的学习曲线
cls_loss: 1.356 → 0.542 (-60.1%)  # 深度学习
mAP50:    0.543 → 0.644 (+18.6%)  # 持续改进

# Phase 9: 停滞的学习曲线
cls_loss: 1.183 → 0.899 (-24.0%)  # 浅层学习
mAP50:    0.507 → 0.572 (+12.8%)  # 早期饱和
```

**诊断**：Phase 9 的分类损失下降幅度仅为 Phase 8 的 **40%**，说明模型对新数据的学习能力严重受限。

### 洞察 2: 异质性的毒性阈值

```python
变异系数 (CV = std / mean):
Phase 8: CV = 12.4 / 12.0 = 1.03  # 正常
Phase 9: CV = 208.1 / 63.1 = 3.30  # 异常

CV 比值: 3.30 / 1.03 = 3.2x
```

**理论依据**：当数据异质性 CV 超过 2x 基准时，深度学习模型会遭遇**梯度方向冲突**，导致收敛困难。

### 洞察 3: 中位数陷阱

```python
Phase 9 标注密度:
mean = 63.1  # 看起来不错
median = 3.0  # ⚠️ 真相！62.7% 图片 <10 标注
```

**教训**：在数据质量评估中，**中位数比均值更能反映真实情况**。Phase 9 被少量高密度图片（8% >100 标注）拉高了均值，掩盖了大量低质量样本。
